# Lorcana Technical Spike

Use when a port layer is uncertain, such as seedrandom compatibility, JSON shape, snapshot schema, card IR generation, or TypeScript/Rust differential harnessing.

A spike must not mutate production runtime behavior.

Required output:
1. Question being answered.
2. Oracle files inspected.
3. Candidate approaches.
4. Recommended approach.
5. Rejected approaches and why.
6. Minimal prototype or experiment path.
7. Tests needed before productionizing.
8. Registry entries needed if accepted.
9. Risks that remain.

End with a go/no-go recommendation.
