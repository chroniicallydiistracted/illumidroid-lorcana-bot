# Section 6: Abilities, Effects, and Resolving

**Comprehensive Rules Version 2.0.1** - Rule References 6.1-6.7

## Overview

This section covers how abilities work, types of abilities (triggered, activated, static), replacement effects, ability modifiers, and the comprehensive rules for resolving cards and effects.

---

## 6.1 General

**Rule refs:** 6.1.1-6.1.8

**Key Concepts:**

- Abilities normally apply only when source is in play
- Source is normally card or effect generating it
- Effects are what happens as a result of resolving action or ability

### Effect Definitions

- **Effect:** What happens as result of resolving action/ability
- **Source:** Card or effect generating ability/effect
- **Player generating effect:** Player who played card or controlled source

### Playing "For Free" (6.1.7-6.1.7.1)

- **Card:** Play without paying ink cost
- **Ability:** Use without paying costs except ○
- Still must follow all other rules

### "For Each" (6.1.8)

- Defines number by counting game elements
- Counted when effect resolves
- Each counted item generates separate instance

## 6.2 Triggered Abilities

**Rule refs:** 6.2.1-6.2.9

Triggered abilities automatically add effects to the bag when conditions are met.

### 6.2.1 Definition

- Continuously look for specific condition
- Have effect when condition met
- Start with: "When", "Whenever", "At the start of", "At the end of"

### 6.2.2 Adding to Bag

- When condition met, ability added to bag
- If met during resolving effect, waits until that effect finishes
- Added by player whose card generated it

### 6.2.3 Delayed Triggered Abilities

- Generated to resolve at specific later moment
- Wait in bag until that moment

### 6.2.4 Secondary "If" Conditions

- In effect text: "If [condition], [effect]"
- Checked during resolution (not when added to bag)
- If false, effect resolves with no effect

### 6.2.5 Triggered Ability Sources

- Can be from character in play
- Can be from card in another zone (if ability specifies)
- Can be from resolved effect

### 6.2.6 "At Start/End of Turn" (6.2.6-6.2.8)

- **At start of your turn:** Added during Ready step, resolve in Set step
- **End of your turn:** Added when declaring end, resolve during End-of-Turn Phase
- **Start/End of turn:** Without "your", work for all players

### 6.2.7 Intervening "If"

- "If/When/Whenever [X], if [Y], do [Z]"
- Y checked when resolving, not when triggering

### 6.2.8 "See" Leaving Play

- When card leaves play, triggered abilities "see" other cards leaving with it
- Affects how many times ability triggers

### 6.2.9 Choices During Resolution

- Choices made during resolution, not when triggered
- Applies to targets, modes, and other decisions

---

## 6.3 Activated Abilities

**Rule refs:** 6.3.1-6.3.3

Activated abilities are used by paying a cost.

### 6.3.1 Definition

- Player chooses to use
- Requires cost payment
- Cost indicated before effect text (usually ○ and/or ink)

### 6.3.1.1-6.3.1.2 Timing Restrictions

- **With ○ on character:** Character must be dry
- **Without ○ on character:** Can use same turn played
- **Items/Locations:** Can use same turn played

### 6.3.2 Using During Turn

- While no effects resolving
- Character not questing or in challenge
- Active player may use as turn action

### 6.3.3 Triggered Abilities from Activation

- Triggered abilities from activating go into bag
- Won't resolve until activated ability's effect resolves

---

## 6.4 Static Abilities

**Rule refs:** 6.4.1-6.4.5

Static abilities continuously alter characteristics or rules.

### 6.4.1 Definition

- Effects that alter characteristics, rules, or game state
- Continuously active for specific duration (or while source in play)

### 6.4.2 Sources of Static Abilities

- **Resolved effects:** Have specified duration
- **Cards in play:** No duration (lasts while source in play)

### 6.4.2.1 Continuous from Resolved Effects

- Affects all cards in play that ability can affect
- Lasts for duration
- Cards played after effect resolves are also affected

### 6.4.2.2 Applied Static Abilities

- Affects only cards in play when effect resolved
- Cards played later are not affected

### 6.4.2.3 Continuous from Cards in Play

- Affects all cards in play
- Lasts while source card in play
- Ends immediately when source leaves play

### 6.4.3 Conditional Static Abilities

- Tie effect and condition with "unless" or "if"
- Check if condition true/false to determine if effect applies

### 6.4.4 Conditional Patterns

- **[Effect] unless [Condition]:** Effect applies while condition false
- **If [Condition], [Effect]:** Effect applies while condition true
- **[Effect] if [Condition]:** Effect applies while condition true

### 6.4.5 Skip Effects

- "Skip [Step/Phase]" means that step/phase doesn't happen
- No abilities or effects from that step/phase happen
- Can't skip if step/phase already started

---

## 6.5 Replacement Effects

**Rule refs:** 6.5.1-6.5.8

Replacement effects replace one event with another.

### 6.5.1 Definition

- Wait for condition to occur
- Partially or completely replace event as it resolves
- Most include word "instead"

### 6.5.2 Event Definition

- Resolution of an effect as a whole
- Multiple effects = one event

### 6.5.3 Applying Replacement

- Must exist and apply when event would happen
- Can't replace already-happened event
- Replacement condition checked at right moment

### 6.5.4 Modified Events

- First event never happens
- Modified event occurs instead
- Can trigger different abilities
- Original triggers don't happen

### 6.5.5 One Chance Only

- Replacement effect has one chance per event
- Can't apply again to same event even if modified

### 6.5.6 Self-Replacement Effects

- Replace own ability/abilities
- Always applied first

### 6.5.7 Multiple Replacement Effects

When multiple can apply:

1. Apply self-replacement if able
2. Choose and apply another
3. Repeat until no more apply

### 6.5.8 Same Replacement Effect

- Two+ instances of same effect can't apply to same event
- Choose one, rest cease to exist

## 6.6 Ability Modifiers

**Rule refs:** 6.6.1-6.6.4

Modifiers change characteristics like Ⓒ, ⣇, or ◇.

### 6.6.1 Continuous Application

- Apply to card continuously
- Apply immediately when they would affect a characteristic
- Don't use bag
- All modifiers combine together

### 6.6.1.2 Combining Modifiers

- Don't apply in specific order
- All combine to apply together
- New modifier combines with existing

### 6.6.2 Negative Ⓒ

- Deals no damage during challenges
- Counts as 0 for most purposes
- Still has negative value for modifier calculations

### 6.6.3 Negative ◇

- Counts as 0 for lore gained
- Still has negative value for modifier calculations

### 6.6.4 "Can't Be Reduced Below"

- Characteristic can't be less than specified value
- Checked after all modifiers applied

## 6.7 Resolving Cards and Effects

**Rule refs:** 6.7.1-6.7.8

Comprehensive process for resolving played cards and effects.

### 6.7.1 Resolving Played Cards

**Characters/Items/Locations (6.7.1.1):**

- Enter Play zone
- If Shift, put on top of same-name card

**Actions (6.7.1.2):**

- Enter Play zone
- Effect resolves immediately
- Put into discard after effect

### 6.7.2 Resolving Effects (in order)

1. **Count "for each"** - Determine number if effect uses "for each" (6.7.2.1)
2. **Check secondary "if"** - If false, resolve with no effect (6.7.2.1)
3. **Check replacement effects** - Apply any that would modify resolving effect (6.7.2.2)
4. **Calculate damage** - If dealing damage, apply modifiers to determine final amount (6.7.2.3)
5. **Perform instructions** - Follow all instructions in order, make choices (6.7.2.4)

### 6.7.3 Triggered Abilities During Resolution

- Added to bag but don't resolve yet
- Wait until current effect fully resolves
- If multiple effects, wait until all done

### 6.7.4 Game State Check

- Happens after every effect resolves

### 6.7.5 Multiple Players Acting

- Active player goes first
- Then proceed in turn order
- Each part is same resolution

### 6.7.6 Last Known Value

- If referencing characteristic of card no longer in play
- Use last known value from before it left play

### 6.7.7 Playing Cards During Resolution

- Card played while effect resolving
- Its effects wait to resolve until initial effect finishes
- Applies to effects of instructed card too

### 6.7.8 Modify How Card Enters Play

- Some effects modify entry
- Resolves along with card being played
- No moment where card is in play without modification

## Quick Reference

| Ability Type    | How Triggered/Used        | When Resolves        | Key Rules                                |
| --------------- | ------------------------- | -------------------- | ---------------------------------------- |
| **Triggered**   | Condition met             | When chosen from bag | Continuous check, added to bag           |
| **Activated**   | Player chooses, pays cost | Immediately          | Must be dry (for ○), not in challenge    |
| **Static**      | Automatically             | Continuously         | Conditional (if/unless) or unconditional |
| **Replacement** | Event would happen        | Instead of event     | One chance, self-replacement first       |

| Modifier Type                | How Applied                                 |
| ---------------------------- | ------------------------------------------- |
| **Ⓒ modifiers**              | Combine, negative = 0 for damage            |
| **◇ modifiers**              | Combine, negative = 0 for lore              |
| **"Can't be reduced below"** | After all modifiers, characteristic ≥ value |
